1 Objetivos

OBJ-0001 Establecer presencia digital profesional para captar alumnos
Versión 1.0 (06/10/2025)
Descripción

El sistema deberá Crear un sitio web que sirva como canal principal de información y captación de alumnos, complementando la estrategia de redes sociales.
Subobjetivos
  • [OBJ-0008] Mostrar oferta de clases: El sistema deberá comunicar profesionalidad, mostrar la oferta de clases y facilitar el proceso de reserva y pago.
Comentarios

Como métricas de éxito se tiene:

· Conseguir 20-30 inscripciones mensuales desde la web

· Reducir consultas repetitivas en redes sociales en un 40%

· Tiempo en sitio superior a 2 minutos
OBJ-0002 Facilitar proceso de reserva y pago previo
Versión 1.0 (06/10/2025)
Descripción

El sistema deberá Permitir que alumnos potenciales puedan reservar clases (especialmente workshops y masterclasses) y realizar pagos mediante transferencia bancaria o tarjeta antes de asistir. No se requiere sistema de reserva en tiempo real, sino formulario que derive a contacto por redes sociales y posterior pago por cuenta bancaria.
Comentarios

Como métricas de éxito se tiene:
·Reducir no-shows en eventos especiales en un 60%

·80% de pagos realizados antes de la clase

·Tiempo de gestión de reservas reducido en 50%
OBJ-0003 Mostrar calendario actualizado de clases y eventos
Versión 1.0 (06/10/2025)
Descripción

El sistema deberá Publicar de forma clara y visual los horarios de clases regulares, workshops, masterclasses y eventos especiales..
Subobjetivos
  • [OBJ-0006] Reserva obligatoria: El sistema deberá destacar aquellos que requieren reserva previa obligatoria..
Comentarios

Como métricas de éxito se tiene:
·Consultas sobre horarios reducidas en 50%

·90% de los visitantes encuentran horarios en menos de 30 ·segundos
Actualización de calendario al menos semanal
OBJ-0004 Promocionar merchandising de la escuela
Versión 1.0 (06/10/2025)
Descripción

El sistema deberá Mostrar el merchandising disponible (camisetas, sudaderas, gorras) como elemento publicitario, sin sistema de compra online..
Subobjetivos
  • [OBJ-0007] Gestionar pedidos: El sistema deberá Los pedidos se gestionarán presencialmente o por Instagram..
Comentarios

Como métricas de éxito se tiene:
·Incremento del 30% en ventas de merchandising

·Mayor visibilidad de marca en eventos
OBJ-0005 Posicionamiento en bísquedas locales
Versión 1.0 (06/10/2025)
Descripción

El sistema deberá Aparecer en los primeros resultados de Google para búsquedas relacionadas con baile urbano en Sevilla, atrayendo tráfico orgánico de alumnos potenciales..
Comentarios

Como métricas de éxito se tiene:
·Ranking top 3 para "clases baile urbano Sevilla"

·Ranking top 5 para "escuela hip hop Sevilla"

·40% del tráfico web desde búsqueda orgánica

3 Requisitos de interfaz de usuario

NFR-0010 Diseño prioritario
Versión 1.0 (10/10/2025)
Descripción

El sistema deberá funcionar perfectamente en dispositivos móviles, tablets y escritorio. Dado que el público objetivo (jóvenes interesados en baile urbano) usa principalmente móvil, el diseño mobile-first es crítico.
Comentarios

Dispositivos objetivo:
·70% móvil (iOS y Android)

·20% escritorio

·10% tablet
NFR-0011 Identidad visual urbana y juvenil
Versión 1.0 (10/10/2025)
Descripción

El sistema deberá tener un estilo visual que refleje la cultura del baile urbano: moderno, energético, juvenil, con elementos street art si es posible.
Comentarios

Urbano, juvenil, dinámico, con uso de fotografías de alta calidad de clases y eventos
NFR-0012 Destacar eventos especiales
Versión 1.0 (10/10/2025)
Descripción

El sistema deberá destacar los workshops, masterclasses y eventos en la home, con banners llamativos y botones de acción visibles.
Comentarios

Deben tener protagonsimo visual ante lo demás
NFR-0013 Estructura de menús simple
Versión 1.0 (10/10/2025)
Descripción

El sistema deberá tener un menú principal claro y accesible con máximo 6-7 elementos.
Comentarios

, política de cookies
, términos y condiciones...
NFR-0014 Botones de acción visibles
Versión 1.0 (10/10/2025)
Descripción

El sistema deberá tener los botones de acción claros y fáciles de ver.

NFR-0015 Integración visual con redes sociales

6 Reglas de negocio

CRQ-0001 Política de pagos previos
Versión 1.0 (10/10/2025)
Descripción

El sistema deberá respetar la siguiente regla de negocio o restricción: Para clases que requieren reserva (workshops, masterclasses), el pago debe realizarse ANTES de asistir. No se permiten reservas sin pago previo.
Comentarios

El formulario de reserva debe indicar claramente que tras enviar datos, deben realizar transferencia y confirmar por Instagram
CRQ-0002 Política de no devoluciones
Versión 1.0 (10/10/2025)
Descripción

El sistema deberá respetar la siguiente regla de negocio o restricción: ·Clases puntuales pagadas: No reembolso si el alumno no asiste
·Mensualidades: Si avisan antes de fin de mes que no continúan, no se cobra el siguiente mes. Si ya pagaron y faltan días, no hay devolución

·Clases canceladas por la escuela: Se reubica al alumno en otra clase u horario.
Comentarios

Debe estar claramente visible en página de precios y términos y condiciones
CRQ-0003 Restricciones de edad
Versión 1.0 (10/10/2025)
Descripción

El sistema deberá respetar la siguiente regla de negocio o restricción: Clases deben estar etiquetadas claramente por rango de edad.
Comentarios

·Kids: 5-11 años

·Youth: 10-15 años (flexible según nivel del niño)

·Adultos: 16+ años
CRQ-0004 Aforo máximo por sala
Versión 1.0 (10/10/2025)
Descripción

El sistema deberá respetar la siguiente regla de negocio o restricción: Sistema de reservas debe controlar aforo (aunque sea manual mediante formulario).
Comentarios

·Sala inferior: Máximo 35 personas

·Sala superior: Máximo 70 personas
CRQ-0005 Clases de prueba gratuitas
Versión 1.0 (10/10/2025)
Descripción

El sistema deberá respetar la siguiente regla de negocio o restricción: Primera vez que un alumno viene a la escuela puede probar 1-2 clases gratuitamente.
Comentarios

Debe mencionarse en home y página de clases como incentivo
Ejemplo: Banner "Primera clase GRATIS para nuevos alumnos"
CRQ-0006 Reserva previa obligatoria
Versión 1.0 (10/10/2025)
Descripción

El sistema deberá respetar la siguiente regla de negocio o restricción: Eventos especiales deben indicar claramente "Reserva previa obligatoria".
Comentarios

·Obligatoria: Workshops, masterclasses, eventos especiales
·
No obligatoria: Clases regulares, pueden venir directamente
CRQ-0007 Recuperación de clases
Versión 1.0 (10/10/2025)
Descripción

El sistema deberá respetar la siguiente regla de negocio o restricción: Clases perdidas por flexibilidad de la escuela: SÍ se pueden recuperar, clases perdidas por ausencia del alumno: NO se recuperan.
Comentarios

Explicar en términos y condiciones
CRQ-0008 Cierre de clases por falta de alumnos
Versión 1.0 (10/10/2025)
Descripción

El sistema deberá respetar la siguiente regla de negocio o restricción: Si tras periodo de prueba (ejemplo: 1 mes) una clase no alcanza el mínimo de alumnos, se cierra y se reubica a los alumnos inscritos en otra clase/horario.
Comentarios

Indicar en términos y condiciones que los horarios pueden estar sujetos a cambios
CRQ-0009 Cumplimiento legal
Versión 1.0 (10/10/2025)
Descripción

El sistema deberá respetar la siguiente regla de negocio o restricción: Consentimiento explícito para tratamiento de datos personales
Información sobre finalidad (gestión de reservas, comunicaciones)
Derecho de acceso, rectificación, supresión
Responsable: Anabel García López / Dadix Dance Studio.

·Autorización de imagen
Checkbox obligatorio en formulario de reserva
Indica que pueden aparecer en fotos/vídeos de la escuela en web y redes sociales
Posibilidad de revocar consentimiento

Añadir campo "¿Es menor de 14 años?" con casilla para datos de padre/tutor
