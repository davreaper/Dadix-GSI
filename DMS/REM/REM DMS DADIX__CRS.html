IRQ-0001 Contrato de matrícula
Versión 1.0 (10/11/2025)
Autores
  • David Real
Descripción

El sistema deberá almacenar la información correspondiente a los contratos de las matrículas de los alumnos. En concreto:
Datos específicos
  • Nombre del alumno
  • DNI del alumno
  • Edad
  • Contacto (email, telefono)
  • Datos del tutor (si es menor)
  • Clases matriculadas
  • Modalidad de pago
  • Mensualidad
  • Fecha de matrícula
Importancia vital

6 Casos de uso

UC-0001 Consultar contrato de matrícula
Versión 1.0 (10/11/2025)
Autores
  • David Real
Descripción

El sistema deberá comportarse tal como se describe en el siguiente caso de uso cuando el actor consulta la ficha médica de un alumno para conocer restricciones médicas.
1-El actor accede al DMS

2-Busca por nombre del alumno

3-Filtra por tipo de documento "Ficha Médica"

4-El sistema muestra la ficha médica

5-El actor revisa alergias, lesiones previas y restricciones
Precondición

El contrato debe existir en el sistema
Postcondición

PD

UC-0003 Verificar Pagos de Alumno
Versión 1.0 (10/11/2025)
Autores
  • David Real
Descripción

El sistema deberá comportarse tal como se describe en el siguiente caso de uso cuando el administrador verifica los pagos realizados por un alumno.
1-El administrador accede al DMS

2-Busca por DNI o nombre del alumno

3-Filtra por tipo de documento "Justificante de Pago"

4-Filtra por periodo (mes/año)

5-El sistema muestra los justificantes correspondientes

6-El administrador verifica el estado de pago
Precondición

Deben existir justificantes de pago registrados.
Postcondición

PD

UC-0005 Buscar alumnos por clase
Versión 1.0 (10/11/2025)
Autores
  • David Real
Descripción

El sistema deberá comportarse tal como se describe en el siguiente caso de uso cuando el actor busca todos los alumnos matriculados en una clase específica

1-El actor accede al DMS

2-Filtra por tipo de documento "Contrato de Matrícula"

3-Aplica filtro por clase (ej: "Comercial - Avanzado")

4-El sistema muestra todos los contratos de esa clase

5-El actor consulta la lista de alumnos
Precondición

Deben existir contratos de matrícula
Postcondición

PD

Versión 1.0 (10/11/2025)
Autores
  • David Real
Descripción

El sistema deberá implementar control de acceso basado en roles:

·Administrador: Lectura/Escritura sobre todos los documentos

·Profesor: Lectura sobre contratos y fichas de sus alumnos
·Alumno/Tutor: Lectura sobre sus propios documentos.
